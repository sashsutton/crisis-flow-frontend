The **CrisisFlow Frontend** is the real-time command center for disaster intelligence. It visualizes the structured insights generated by the AI Engine, presenting chaotic text streams as actionable geospatial and semantic data.

The frontend is built with **React** and consumes endpoints exposed by the CrisisFlow AI Engine.

- **Backend Repository:** [GitHub - CrisisFlow AI Engine](https://github.com/sashasutton/crisis-flow-backend)
- **Live Website:** [CrisisFlow Dashboard](https://crisis-flow-frontend-mhy4qore7-sashasuttons-projects.vercel.app/)

## ğŸ—ï¸ Architecture

The Frontend application is structured to visualize data in two primary modes:

### 1) Geospatial View (Map)
Renders incidents on an interactive map using `react-leaflet`.
- **Markers**: Colored by semantic cluster (e.g., Red for Critical Rescue).
- **Popups**: Display incident details and category.

### 2) Neural Cluster View (Charts)
Visualizes the semantic similarity of messages using `recharts`.
- **Scatter Plot**: Plots 2D PCA coordinates (`pca_x`, `pca_y`) received from the AI Engine.
- **Clusters**: Groups similar messages to show emerging patterns.

### 3) Live Feed
Displays a real-time stream of incoming messages with their assigned categories.
